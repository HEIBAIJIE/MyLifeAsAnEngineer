# 《我的工程师生活》游戏指南

## 🎮 游戏简介

《我的工程师生活》是一个深度的生活模拟游戏，让你体验真实的工程师职场生活。通过感性和理性两条不同的路径，探索生活的本质和意义。

## 🚀 快速开始

### 安装和运行
```bash
# 安装依赖
npm install

# 启动游戏
npm run game

# 或者直接运行
npx ts-node start-game.ts
```

### 测试连接
```bash
# 测试前端与后端连接
npm run test-frontend
```

## 🎯 游戏目标

- **感性路径**：通过社交、艺术、直觉和情感的力量来理解生活
- **理性路径**：通过逻辑、科学、分析和思辨来理解生活
- **平衡发展**：在感性和理性之间找到平衡，探索生活的真谛

## 🕹️ 操作指南

### 基本操作
- **数字1-6**：快速切换场景
  - 1 - 公司（工作场所）
  - 2 - 商店（购买物品）
  - 3 - 家（休息恢复）
  - 4 - 公园（锻炼思考）
  - 5 - 餐馆（用餐社交）
  - 6 - 医院（治疗健康）

- **数字7+**：执行对应编号的事件
- **字母命令**：
  - `s` - 保存游戏进度
  - `l` - 读取游戏进度
  - `i` - 查看物品栏
  - `h` - 显示帮助
  - `q` - 退出游戏

## 📊 属性系统详解

### 基础属性
- **💰 金钱**：购买物品和服务的货币
- **❤️ 健康**：身体健康状况，影响工作效率
- **😴 疲劳**：身体疲劳程度，需要休息恢复
- **🍽️ 饥饿**：饥饿程度，需要进食恢复

### 核心属性
- **🧠 理性**：逻辑思维能力，影响技术工作效率
- **💖 感性**：情感直觉能力，影响社交工作效率
- **🎯 专注**：执行困难任务的意志力资源
- **😊 心情**：当前情绪状态，影响各种活动效果

### 职业属性
- **🔧 技能**：专业技术能力水平
- **👔 职级**：职业等级（1-10级）
- **📊 项目**：当前项目完成进度
- **😠 老板**：老板对你的不满程度

### 高级属性
- **🤝 社交影响力**：通过社交积累的隐形资源
- **🏆 技术声誉**：通过技术工作积累的声誉
- **🤔 哲学感悟**：通过思考积累的人生感悟

## ⏰ 时间系统

- **时间单元**：每天48个时间单元，每个单元30分钟
- **工作日**：周一至周五，可进行工作相关活动
- **周末**：周六周日，工作事件受限，适合休息
- **白天/夜晚**：影响某些活动的效果

## 🏢 场景系统

### 公司
- **工作事件**：编程、设计、会议等
- **社交事件**：与老板、同事互动
- **学习事件**：技能提升、知识学习

### 商店
- **购买物品**：生活用品、工作装备
- **技术书籍**：提升专业技能
- **哲学书籍**：增加人生感悟

### 家
- **休息恢复**：睡眠、放松
- **个人学习**：读书、思考
- **生活管理**：整理、规划

### 公园
- **锻炼身体**：提升健康度
- **思考人生**：增加哲学感悟
- **放松心情**：缓解压力

### 餐馆
- **用餐**：恢复饥饿度
- **社交**：与他人交流
- **享受生活**：提升心情

### 医院
- **治疗**：恢复健康
- **体检**：了解身体状况
- **咨询**：获得健康建议

## 🎲 事件系统

### 事件分类
- **快速行动**（≤1小时）：简单的日常活动
- **中等行动**（1-2.5小时）：需要一定投入的活动
- **长时间行动**（>2.5小时）：深度投入的重要活动

### 事件效果
- **资源变化**：各种属性的增减
- **临时事件**：随机触发的特殊情况
- **定时任务**：延迟生效的持续影响

## 💾 存档系统

- **保存游戏**：输入`s`生成存档代码
- **读取游戏**：输入`l`并提供存档代码
- **存档代码**：Base64编码的游戏状态，请妥善保存

## 🎯 游戏策略

### 新手建议
1. **平衡发展**：不要只专注于工作，注意休息和娱乐
2. **时间管理**：合理安排工作日和周末的活动
3. **资源管理**：注意疲劳、饥饿等基础需求
4. **社交投资**：与同事和老板保持良好关系

### 进阶策略
1. **路径选择**：决定走感性路径还是理性路径
2. **职业规划**：通过技能提升和项目完成获得晋升
3. **哲学思考**：通过深度思考获得人生感悟
4. **生活平衡**：在工作成就和生活质量间找到平衡

## 🔧 故障排除

### 常见问题
1. **游戏无法启动**：检查Node.js和TypeScript安装
2. **显示异常**：确保终端支持UTF-8编码
3. **命令无响应**：重启游戏或检查输入格式

### 调试模式
如需调试，可以在代码中添加日志输出：
```typescript
console.log('Debug: 当前状态', gameState);
```

## 🎨 自定义和扩展

游戏支持通过修改CSV文件来自定义内容：

- `csv/resources.csv` - 资源配置
- `csv/locations.csv` - 场景设置
- `csv/items.csv` - 物品数据
